# 6.2.3 Entitlement Descriptions and Metadata

**Learning Objective:** Understand how to add context and metadata to entitlements to make them meaningful and manageable.

---

## What is Entitlement Metadata?

**Definition:** Additional information and context added to entitlements beyond what is discovered during aggregation

**Key Concept:**
```
Discovered vs. Enhanced:

Discovered (from aggregation):
  - Name: VPN-Users
  - Value: CN=VPN-Users,OU=Groups,DC=company,DC=com
  - Source: Active Directory
  - Members: 450
  
Enhanced (metadata added by admin):
  - Description: "Remote access to corporate network"
  - Owner: IT Operations Team
  - Risk Level: Medium
  - Business Purpose: "Work from home capability"
  - Requestable: Yes
  - Approver: Manager
  - Compliance: SOX controls apply
```

**Why Metadata Matters:**
```
Without metadata:
  User sees: "VPN-Users"
  User thinks: "What is this? Should I request it?"
  Result: Confusion, wrong requests
  
With metadata:
  User sees: "VPN Access - Remote access to corporate network"
  User thinks: "I need this for working from home"
  Result: Clear understanding, correct requests
```

**Exam Tip:** Metadata makes entitlements understandable to users and manageable for admins. Always add descriptions and business context.
